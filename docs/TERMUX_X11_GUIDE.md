# 🖥️ Termux:X11 Guide ## Scope This guide explains how `Termux:X11` is used as the Android-side display server for the Debian desktop, and how to troubleshoot common issues. ## Startup Flow 1. Host ensures `termux-x11` is present. 2. `mount-debian.sh` exposes hardware nodes and runtime dirs. 3. `startxfce4_chrootDebian.sh` starts the clipboard sync daemon and launches XFCE inside the chroot. 4. `Termux:X11` binds to `:0` on Android. ## Common Fixes - Black screen: stop chroot, delete `.X0-lock` and socket, restart. - Permission errors: ensure `/run/user/1000` exists and is owned by UID 1000. - Input lag: avoid `picom` on low-end devices; disable compositing via `user-session.sh`. ## Related Docs - [docs/GPU_ACCELERATION.md](../docs/GPU_ACCELERATION.md) - [docs/DEBIAN_DESKTOP_GUIDE.md](./DEBIAN_DESKTOP_GUIDE.md) - [ARCHITECTURE.md](../ARCHITECTURE.md)
