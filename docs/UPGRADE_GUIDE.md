# ⬆️ Upgrade Guide ## Scope This guide describes how to upgrade an existing `Systemless-Host-Termux-SU` installation to a newer release while preserving your chroot data, user home, and custom configurations. It applies to hosts running Termux on rooted Android and a Debian chroot at `/data/local/tmp/chrootDebian`. ## Assumptions - The canonical guest user is `Ruusian5`. - The chroot path is `/data/local/tmp/chrootDebian` unless overridden by environment variables. - Existing host credentials (Wi-Fi, Termux setup) are valid. ## Pre-Upgrade Checklist 1. Ensure the host has enough free storage for the update payload. 2. Back up the guest home directory before touching the chroot. 3. Confirm root/`su` access still works. 4. Record the current repo **origin** in case you need to re-sync: ```bash git remote -v ``` ## Upgrade Steps ### 1. Update Repo Files Enter the repo and fetch the latest code: ```bash cd ~/Systemless-Host-Termux-SU git pull origin main ``` If you installed from a fork or alternate branch, replace `main` and `origin` accordingly. ### 2. Re-run the Installer The installer is designed to be idempotent and preserve existing config. It will: - Refresh Termux host dependencies only if missing or outdated. - Re-deploy dashboard scripts and shell integration. - Re-apply kernel mounts and guest hardening. Run: ```bash bash install.sh ``` ### 3. Validate Guest Environment Confirm the chroot mounts and user account: ```bash su -c "test -d /data/local/tmp/chrootDebian/home/ruusian" && echo OK ``` Verify the offline toolkit still resolves paths: ```bash cd offline-toolkit ./verify.sh checksums.sha256 ``` ### 4. Rollback (Optional) If the upgrade misbehaves: ```bash git checkout HEAD~1 ./install.sh bash install.sh ``` Then restore the chroot from backup using `offline-toolkit/restore.sh`. ## Post-Upgrade Verification - `agy` shows updated dashboard UI (modern HUD / Updated nods). - `res` correctly toggles resolution. - `gpu` audit reports the expected KGSL/VirGL state. - No duplicate `.bashrc` entries for aliases or `termux-wake-lock`. ## Related Docs - [INSTALL.md](INSTALL.md) for first-time setup - [docs/TEST_PLAN.md](../docs/TEST_PLAN.md) for regression checks - [offline-toolkit/BACKUP.md](../offline-toolkit/BACKUP.md) for backup flows
