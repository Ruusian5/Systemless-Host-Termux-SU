# 🚑 Recovery Guide ## Scope This guide covers recovery procedures for `Systemless-Host-Termux-SU` when a Termux host package, the Debian guest, or a launcher/script fails unexpectedly. The canonical guest user is `Ruusian5` and the default chroot path is `/data/local/tmp/chrootDebian`. ## Prerequisites - Root / `su` access working - The repo present at `~/Systemless-Host-Termux-SU` - Offline bundle (`debian-rootfs.tar.zst`) if you need to rebuild the guest ## Quick Recovery Matrix | Failure mode | First action | Secondary action | | --- | --- | --- | | `agy` broken or aliases missing | Re-run `install.sh` | Restore `.bashrc` from backup | | Debian chroot corrupted | Restore from offline bundle | Re-extract rootfs + rerun installer | | Termux:X11 black screen | `fix` then reboot | Stop chroot + remove `.X0-lock` | | GPU black screen | Set `LIBGL_ALWAYS_HW=0` | Use software fallback temporarily | | Audio crackle / no audio | Restart `pulseaudio` | Re-run `mount-debian.sh` | ## Step-by-Step Recovery ### 1. Restore Host Shortcuts ```bash cd ~/Systemless-Host-Termux-SU bash install.sh ``` This is safe to re-run; it refreshes aliases and scripts without wiping the chroot. ### 2. Rebuild Kernel Bridges ```bash bash ~/Systemless-Host-Termux-SU/scripts/mount-debian.sh ``` Confirm bridges: ```bash mount | grep chrootDebian ``` ### 3. Restore Debian Guest from Offline Bundle 1. Stop services: ```bash bash ~/stop-debian.sh ``` 2. Restore: ```bash cd ~/Systemless-Host-Termux-SU/offline-toolkit ./restore.sh debian-rootfs.tar.zst ``` 3. Re-run host installer script sync: ```bash cd ~/Systemless-Host-Termux-SU bash install.sh ``` ### 4. Emergency User Reset If the guest user state is broken: ```bash su -c "/data/data/com.termux/files/usr/bin/busybox chroot /data/local/tmp/chrootDebian /usr/bin/sh -c ' groupadd -g 1000 Ruusian5 || true useradd -u 1000 -g 1000 -d /home/ruusian -s /bin/bash Ruusian5 || true chown -R 1000:1000 /home/ruusian '" ``` ## Logs - Host Termux logs: `~/logs/` - X11 logs: `~/.termux/x11/` (if present) - Guest session log: `/home/ruusian/session_debug.log` ## Related Docs - [INSTALL.md](../INSTALL.md) - [docs/UPGRADE_GUIDE.md](../docs/UPGRADE_GUIDE.md) - [offline-toolkit/BACKUP.md](../offline-toolkit/BACKUP.md) - [offline-toolkit/RECOVERY.md](../offline-toolkit/RECOVERY.md)
